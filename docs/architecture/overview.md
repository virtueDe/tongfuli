# Tongfuli 总体架构设计

## 1. 设计目标

Tongfuli 不是“聊天框套大模型”的外壳产品，而是一个以《武林外传》内容资产为真源、以角色化 AI 对话为核心交互、以前后端与知识治理平台共同支撑的企业级 IP 对话系统。

本设计追求三件事：

- 角色体验要像：用户感知到的是“这个角色真的在说话”。
- 事实依据要稳：剧情、人物关系、台词出处要能追溯和纠偏。
- 平台治理要强：内容、模型、审核、实验、评测与成本都要可观测、可回滚。

## 2. 总体架构

推荐架构为：

- 模块化单体业务后端
- 独立 AI 编排服务
- 独立知识处理与审核发布平台
- Web 与小程序双 BFF
- 多存储协同的数据底座

逻辑分层如下：

1. 客户端层
   - Web 主站
   - 小程序
   - 后台管理台
2. 接入层
   - Web BFF
   - Miniapp BFF
   - 鉴权、限流、灰度、实验开关
3. 业务应用层
   - 用户与身份域
   - 会话域
   - 角色域
   - 内容域
   - 来源治理域
   - 运营域
   - 审核与风控域
4. AI 能力层
   - 问题理解
   - 检索编排
   - Prompt 策略
   - 模型路由
   - 结果后校验
5. 数据处理层
   - 剧本导入
   - 网页抓取
   - 清洗、切分、抽取、去重、打标
   - 审核发布与索引刷新
6. 数据与基础设施层
   - PostgreSQL
   - Redis
   - OpenSearch
   - pgvector
   - Neo4j
   - 对象存储
   - Kafka
   - Temporal
   - OpenTelemetry 全链路观测

## 3. 核心边界

### 3.1 主业务系统

主业务系统负责“业务真相”，不负责大模型推理细节。建议采用 `Java 21 + Spring Boot 3 + Spring Modulith`。

模块建议：

- `identity-module`
- `conversation-module`
- `character-module`
- `content-module`
- `source-governance-module`
- `operation-module`
- `analytics-module`
- `moderation-module`
- `admin-console-module`

### 3.2 AI 编排服务

建议采用 `Python 3.12 + FastAPI` 独立部署，负责：

- 问题分类与路由
- 模式与角色约束拼装
- 多路检索与重排
- 模型调用与流式返回
- 后校验与降级处理

### 3.3 知识处理平台

负责把原始剧本和外部资料加工成可上线知识。职责包括：

- 文档导入与版本登记
- 网页抓取与来源治理
- 清洗、切分、抽取、去重
- 审核工作流
- 发布态知识快照生成
- 检索索引重建

## 4. 知识架构

知识系统必须三层分离：

1. `Canonical`
   - 原始剧本
   - 官方可核实资料
   - 用于事实问答、出处校验、关系判断
2. `Interpretation`
   - 人物解读
   - 剧情分析
   - 梗解释
   - 用于增强可读性和解释性
3. `Entertainment/Fanon`
   - 二创表达
   - 现代梗延展
   - 只在扩展或娱乐模式中增强

任何线上回答都必须只读取“已发布快照”，不能直接读未审核草稿。

## 5. 对话系统设计

### 5.1 会话模型

- 一个 `Session`
- 多个 `Turn`
- 每个 `Turn` 显式绑定当前角色、当前模式、问题类型、命中的知识层级、回答置信度与风控结果

### 5.2 模式系统

- `原剧模式`
- `扩展模式`
- `娱乐模式`

模式不是一段 prompt，而是对检索权重、角色边界、自由度和后校验阈值的组合配置。

### 5.3 问题路由

- 事实问答
- 角色闲聊
- 梗互动
- 越界话题
- 混合问题

回答原则必须遵循“先事实、后表演”。

## 6. 前后台形态

- 前台：首页主对话页、角色切换抽屉、模式切换入口、热门问题与梗入口、来源弱展示与分享落地页。
- 后台：内容资产工作台、来源治理工作台、AI 策略工作台、问答诊断工作台、风控与审核工作台。

## 7. 技术选型

- 前端：`Next.js 15 + React 19 + TypeScript + Tailwind CSS + shadcn/ui`
- 小程序：`Taro + React + TypeScript`
- 主业务：`Java 21 + Spring Boot 3 + Spring Modulith`
- AI/数据处理：`Python 3.12 + FastAPI`
- 工作流与队列：`Temporal + Kafka`
- 数据：`PostgreSQL + Redis + OpenSearch + pgvector + Neo4j + 对象存储`

## 8. 非功能目标

- 核心对话链路月可用性：`99.9%`
- 后台与批处理链路：`99.5%`
- 首 token 目标：`2.5s - 4s`
- 中等复杂度完整回答：`6s - 12s`
- 流量规模按 `1 万 - 10 万 DAU` 设计

一句话总结：Tongfuli 的正确实现方式，是把《武林外传》做成一个可计算、可治理、可持续优化的内容世界，而不是把剧本塞进向量库以后碰碰运气。
