# Tongfuli 快速验证指南

本指南用于在项目脚手架搭建完成后，快速验证设计是否被正确落地。验证覆盖主成功路径、一个边界场景和一个失败场景。

## 1. 主成功路径：匿名用户直接开聊并切换角色

### 前置条件

- Web 前端已启动
- 主业务服务、AI 编排服务、检索组件可用
- 至少导入一批已发布的《武林外传》剧本知识快照

### 步骤

1. 打开 Web 首页。
2. 确认无需登录即可看到主对话区。
3. 输入问题：`老白为什么怕佟掌柜？`
4. 观察回答是否满足：
   - 有角色化表达
   - 回答内容与剧中关系一致
   - 支持展开“依据”入口
5. 在当前会话内将角色切换为 `郭芙蓉`。
6. 继续追问：`那你怎么看老白这人？`
7. 观察回答是否满足：
   - 回答角色已经切换
   - 历史消息仍保留
   - 角色口吻和第一轮不同

## 2. 边界场景：剧外问题回拉

1. 将模式切换到 `娱乐模式`。
2. 输入问题：`如果你活在今天，会不会去开直播？`
3. 观察系统是否可以简短接住问题，但最终回到《武林外传》语境。

## 3. 失败场景：低可信外部资料阻断发布

1. 在后台新增一个来源评级为 `low` 的外部站点资料。
2. 执行预处理与抽取。
3. 不进行人工终审，直接尝试发布。
4. 观察系统是否阻断操作并给出原因。

## 4. 通过标准

- 首页可匿名首问
- 角色切换成功
- 两轮回答存在明确风格差异
- 娱乐模式下不会完全变成普通现代聊天机器人
- 低可信未终审资料无法发布
